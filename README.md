# System Info Plugin - 系统信息插件

用于查询 KiraAI 系统状态、配置信息和插件管理的插件。

## 功能特性

- **系统状态查询**：获取操作系统、Python版本、运行时间、CPU、内存、存储等硬件信息
- **项目资源监控**：显示当前进程的内存占用（RSS/VMS）
- **插件信息管理**：查看已加载的插件列表及其启用状态
- **配置信息查看**：查看模型、提供商等配置摘要
- **双模式调用**：支持 `/sysinfo` 指令和 LLM 工具调用两种方式
- **合并转发消息**：支持以合并转发（卡片）形式输出，更美观且不刷屏

## 调用方式

### 方式一：指令调用

发送以下指令直接获取系统信息：

| 指令 | 说明 |
|------|------|
| `/sysinfo` | 获取基础系统信息 |
| `/sysinfo basic` | 获取基础系统信息 |
| `/sysinfo detailed` | 获取详细系统信息（包含进程ID、工作目录、CPU使用率等） |

### 方式二：自然语言调用

直接向机器人询问，LLM 会自动调用工具：

- "查看系统状态"
- "系统运行多久了？"
- "有哪些插件？"
- "查看配置信息"
- "CPU使用率多少？"
- "内存还剩多少？"

## 工具列表

### 1. system_info - 系统状态查询

获取系统运行状态、硬件信息、资源占用等。

**参数**：
- `detail_level`：信息详细程度
  - `basic`：基础信息（默认）
  - `detailed`：详细信息（包含进程ID、工作目录、CPU使用率、最大频率等）

**返回示例**：
```
【系统状态信息】

【基础信息】
系统: Windows 10
Python: 3.10.11
运行时间: 6h 30m
CPU: 16线程 / 8核心 @ 3200MHz
内存: 6.4GB / 16GB (40% 已用)
存储: 120GB / 500GB (24% 已用)
项目占用: 245MB (RSS) / 380MB (VMS)

【插件信息】
总数: 8
启用: 5 | 禁用: 2 | 未配置: 1

【配置信息】
模型数: 2
提供商数: 1

更新时间: 2026-03-23 10:30:00
```

### 2. plugin_list - 插件列表查询

获取当前加载的所有插件列表及其状态。

**参数**：
- `status_filter`：状态过滤
  - `all`：显示所有插件（默认）
  - `enabled`：仅显示已启用的插件
  - `disabled`：仅显示已禁用的插件

**返回示例**：
```
# 插件列表 (5个)

- **system_info_plugin** (enabled)
- **ban_notice_blocker** (enabled)
- **reminder_plugin** (enabled)
- **kiraos_plugin** (disabled)
- **search_plugin** (disabled)

*更新时间: 2026-03-23 10:30:00
```

### 3. config_info - 配置信息查询

查看系统配置信息摘要。

**参数**：
- `section`：配置部分
  - `all`：显示所有配置（默认）
  - `bot`：机器人配置
  - `models`：模型配置
  - `providers`：提供商配置

## 配置选项

编辑 `data/config/plugins/system_info_plugin.json`：

| 配置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| `enabled` | boolean | `true` | 是否启用插件 |
| `verbose_log` | boolean | `true` | 是否输出详细调试日志 |
| `command_prefix` | string | `/` | 指令前缀符号 |
| `show_full_config` | boolean | `false` | 是否显示完整配置（可能包含敏感信息） |
| `max_plugins_list` | integer | `50` | 插件列表最大显示数（10-200） |
| `include_env_vars` | boolean | `false` | 是否包含环境变量信息 |
| `use_forward_message` | boolean | `true` | 是否使用合并转发消息（卡片形式）输出 |

## 合并转发消息

当 `use_forward_message` 为 `true` 时，系统信息会以合并转发消息（卡片）形式发送：

**优点**：
- 更美观的展示效果
- 不会刷屏
- 每个信息模块显示为独立节点

**节点结构**：
1. **系统信息** - 操作系统、Python版本、运行时间、CPU信息
2. **资源监控** - 内存、存储、项目占用
3. **插件管理** - 插件总数和启用数量
4. **配置详情** - 模型数、提供商数、详细信息

**注意**：需要适配器支持合并转发消息，如不支持会自动回退到普通文本消息。

## 安装方法

1. 将 `system_info_plugin` 文件夹放入 `data/plugins/` 目录
2. 重启 KiraAI 或在 WebUI 中刷新插件
3. 插件将自动加载默认配置

## 依赖说明

- **psutil**：用于获取详细的系统信息（CPU、内存、存储、进程占用）
  - 如果未安装，相关功能会被跳过
  - 安装方法：`pip install psutil`

## 日志说明

插件日志分为两个级别：

- **INFO 级别**：启动/卸载、拦截操作等关键事件
- **DEBUG 级别**：详细调试信息（`verbose_log=true` 时输出）

启动日志示例：
```
[SystemInfo] 初始化完成 | 启用:True | 详细日志:True | 合并转发:True
```

拦截日志示例：
```
[SystemInfo] 已卸载
```

## 注意事项

1. **敏感信息**：`show_full_config` 设为 `true` 时可能显示 API Key 等敏感信息，请谨慎使用
2. **性能影响**：获取 CPU 使用率会有约 0.1 秒的延迟
3. **权限要求**：某些系统信息可能需要相应权限才能获取
4. **合并转发**：需要适配器支持，不支持时自动回退到普通文本
